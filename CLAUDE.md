## 3개 전략 정의 (시뮬레이터·제안서 공통)
시뮬레이터는 3개를 **개별·복수·전체** 선택 가능하며, 각 전략마다 **복제 비율(%)**(노셔널 배분, 합 100% 불필요·레버리지 가능)을 입력한다. 비율 w는 전략 함수(가치·델타·페이오프·손익·프리미엄·그릭스)에 곱해져 이론가·차트·MC·합성까지 자동 반영된다(예: 풋 50% + KO 50% + 스프레드 100%). 펀드 보유(복제 대상) 관점의 페이오프로 표현한다.
1. **① 풋 매도 복제 (Short Put)** — OTM 풋을 매도한 페이오프를 복제. 보유가치 V=−Put, 델타 = +(1−e^{−qτ}N(d₁))>0 (주가↓→현물비중↑ = Buy Low). **행사가 K를 S₀ 대비 %로 직접 지정**한다(목표 프리미엄 역산 개념은 사용하지 않음). 지표 "프리미엄 확보확률"은 만기에 풋이 사실상 OTM으로 끝나 수취 프리미엄을 지키는 확률(세전).
2. **② Knock-Out 풋 매수 (Down-and-Out Put)** — 배리어 H(<K) 하향 터치 시 소멸하는 다운아웃 풋 매수 + **리베이트**(낙아웃 시 지급, 만기지급 기준; 입력 %, S₀ 대비). 저렴한 하방보험. 배리어 근처 델타 급변(헤지 난도↑). **만기 손익은 경로의존**:
   - **KO 단일 카드**는 단일 곡선(만기 기초자산 기준: **S≤H → 리베이트, S>H → 생존 페이오프 max(K−S,0)**).
   - **합성 포지션 카드**만 **도달(낙아웃)/미도달(생존)** 2곡선 + 경로의존 설명 노트(미도달 곡선은 S≥최고배리어 구간만).
   - 이론가 = 다운아웃 풋 + 리베이트 현가(R·e^{−rτ}·P(배리어 도달), first-passage 확률). MC에는 **낙아웃 비율** 지표 제공.
3. **③ 스프레드 매수 (Vertical Spread)** — 매수 K₁ / 매도 K₂(K₁<K₂)의 수직 스프레드. **콜**(상승참여+상단cap, 차이나RCF형) 또는 **풋**(하락방어) 선택. 손익 제한.

## 시뮬레이터 수학 규약 (수정 시 필수 준수)
`tools/옵션복제_시뮬레이터.html` 내부 규약:
- **옵션 타입 코드는 `'c'`(콜)/`'p'`(풋)** 이다. `bs(type,...)`가 이 값을 기대하므로 UI select 값도 `c`/`p`여야 한다. (`'call'`/`'put'` 쓰면 전부 풋으로 처리되는 과거 버그 있었음.)
- **이론가**: 유러피언은 Black-Scholes 해석식(가격·Δ·Γ·ν·Θ·ρ). 배리어(다운아웃 풋)는 **Reiner-Rubinstein "아웃 = 유러피언 − 인"**. 배리어 그릭스는 중심차분(FD); vega/rho는 대시(—) 표기.
- **델타(복제 헤지비중)**: 유러피언·스프레드는 해석적, 배리어는 FD.
- **몬테카를로 Dynamic Hedging**:
  - GBM 경로, 위험중립 드리프트 (r−q) 기본(실물 μ 옵션). 시드 고정으로 재현 가능(mulberry32+Box-Muller).
  - 복제 포트폴리오: 시작가치=이론가 V₀, 현물 Δ단위 보유+현금. 매 스텝 현금 무위험성장 **+ 보유현물 배당수익(q) 반영**, 리밸런싱 시 거래비용(bps) 차감.
  - **트래킹오차 = (복제포트 + 누적거래비용) − 실제페이오프** → 거래비용을 제외한 **순수 트래킹오차(평균≈0)**로 헤지 정합성 검증. 거래비용은 별도 지표.
  - **MC 복제가치 ≈ 이론가(BS)** 여야 정합. 배리어는 **일별 모니터링(MC) > 연속 모니터링(클로즈드폼)**이라 다운아웃 값이 약간 큰 것이 정상(이산 배리어 편의).
  - **부호**: 숏풋의 보유가치·MC복제가치는 음수. UI에서 "수취 프리미엄"은 +로 표기.
  - **프리미엄 확보확률**: 만기 세전 페이오프 수익률이 수취 프리미엄에 도달할 확률(풋 사실상 OTM 만기). 거래비용 차감 전 기준.
  - **S₀ 기본값 = 100** (지수/주가의 상대값 기준). 행사가·배리어는 S₀ 대비 %로 입력.
  - **제안서용 대형 산점도(RCF 형식)**: 하단 전체폭. **X=만기 기초자산 가격(S₀=100 기준), Y=동적헤지 실현수익률(%), 점 1개=몬테카를로 경로 1개**, 초록/빨강=손익부호, 노란선=구간평균. 동적헤지 수익률 `dhRet = 이론손익(pnlUnit) − 거래비용 + 복제오차(경로의존)`. **배리어 전략이 있으면 ①미터치(생존)·②터치(낙아웃) 두 산점도로 분리**해서 그린다(경로별 `touched` 플래그). **터치 즉시 KO 델타=0(기능정지)로 청산**(리밸런싱 주기 무관)하므로 터치 경로는 숏풋 하방이 노출돼 큰 손실로 나타난다. 제안서 수익구조 슬라이드의 근거 그림.
  - **최대 편입비 상한(mc.maxW, 기본 180%)**: 총 편입비=Σ전략 델타(=주식 보유비중). 배리어 근처에서 KO 델타가 폭증(예: 총 240%+)해 비현실적이므로, 총 델타 절대값이 상한을 넘으면 **전 전략을 비례 축소**해 캡을 건다(불완전 헤지→트래킹오차 반영). MC 설정에 입력 노출.
  - **산점도 점 색**: 미터치=초록, 터치=빨강, 단일전략=파랑의 **그룹 단색**(손익부호별로 색을 나누지 않음). 두 산점도 **X축 고정 [50,160]=−50%~+60%**로 동일 스케일.
  - **아웃라이어 진단(하단)**: 동적헤지 실현수익률이 이론 손익(payoffRet)에서 벗어난 경로를, 최대이탈 Top5가 아니라 **서로 다른 유형 5가지의 대표 경로**로 뽑는다(보합권 이탈 / 배리어 근처 미터치·델타캡 / 터치·이론 하회 / **터치인데 이론 상회** / 대형 변동). 유형이 없으면 전체 최대이탈로 채움. 각 카드는 **좌측 그래프(2/3폭)** = 한 그래프에 X=시점(개월)·Y=수익률(%)로 **지수 수익률(파랑)·동적헤지 수익률(초록)·이론 손익(노랑)**, **우측(1/3폭)** = 이탈 사유 자동 설명(경로 최저/최고 pathMin·pathMax, 터치 여부, 편입비 캡 등 근거). 전 경로 경량 추적(traceS/traceRet, nSamp≈41)으로 산출.
  - **설정 변경→[실행] 흐름**: 상단 입력 변경 시 이론가는 즉시 갱신, MC·산점도는 [시뮬레이션 실행] 클릭 시 `runAll`이 현재 입력으로 재계산(renderMC+renderScatter).
  - **기본값**: S₀=100, T=1년, **σ50%**, r3%, q2%, 3전략 모두 ON, 풋 K=100·**KO K=100/H=60**/리베이트0·**콜스프레드 K1=110/K2=140**, 비율 각 100%, **MC 5000 paths, 최대 편입비 180%**. 입력에 `autocomplete=off`(새로고침 시 기본값 유지).
  - **애널리틱 차트는 "만기 손익 P&L"(= 페이오프 − 프리미엄×e^{rT} = pnlUnit)** 로 그린다(내재가치가 아니라 프리미엄 포함). 그래서 숏풋은 상단 +프리미엄 평탄, "풋 매도 + 동일행사가 KO 풋 매수"는 미도달 시 (풋프리미엄−KO프리미엄) **상수**가 된다(행사가가 다르면 스프레드 모양). 축은 원금(S₀) 대비 %.
- 검증: `python -m http.server`(아래 실행법) → 브라우저 프리뷰에서 **콘솔 오류 0**, 좌(이론가)·우(MC) 카드 렌더, 이론가↔MC 복제가치 근접 확인.

## 제안서 작성 워크플로우
1. **시뮬레이션** — 시뮬레이터로 변수(만기·σ·r·q·행사가·배리어)와 3전략을 조합, 이론가↔MC 정합 및 손익분포·목표확보확률·VaR을 비교하여 **최적 구조 선정**(사용자와 협의).
2. **백테스트** — 선정 구조를 KOSPI200 과거 시계열로 복제 백테스트 → 목표달성률·회차통계·대표 사례(달성/미달) 산출. **제안서 수익률 구조는 이 백테스트에서 나온다.**
3. **마케팅 포인트** — 사용자가 별도로 제공한다(임의 창작 금지).
4. **PPTX 생성** — **`변동성매매자문_20260513.pptx` 형식/슬라이드 구성을 근간**으로 작성. 최근 파일의 슬라이드 흐름: 표지 → 상품개요(랩 disclaimer) → 투자포인트 → 전략소개(페이오프/편입비/일별프로세스/장단점·이론) → 시뮬레이션·성과(트랙레코드/사례) → 운용부서 소개. 원본은 읽기전용으로 참조만.
   - 컴플라이언스 문구(랩계약 예금자보호 제외, 원금손실 0~100%, 설명의무 등)와 "상기 자료는 이해를 돕기 위한 예시" 캡션을 유지한다.
   - 출처 표기: 시뮬레이션은 본 시뮬레이터, 지수는 실측 출처.
