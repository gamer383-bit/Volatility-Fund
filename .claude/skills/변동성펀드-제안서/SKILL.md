# 변동성 옵션복제 펀드 — 시뮬레이터 & 제안서 스킬

이 프로젝트의 상세 규약은 프로젝트 루트 `CLAUDE.md`에 있다. **먼저 `CLAUDE.md`를 읽고** 이 스킬을 수행한다.
아래는 작업별 절차 요약이다.

## 작업 0 — 최종 버전 기준 작업 (모든 작업 전 필수)
사용자는 **직접 파일을 수정·저장**하기도 하고(로컬 또는 다른 세션→GitHub), Claude에게 요청하기도 한다.
**항상 "최종 버전"을 기준으로 작업한다:**
1. 작업 시작 전 `git fetch` + `git status`로 원격/로컬 상태 확인. 원격이 앞서 있으면 **pull 먼저**, 로컬 미커밋 변경이 있으면 그것이 **사용자의 직접 수정 = 최종본**이므로 되돌리거나 덮어쓰지 않는다.
2. 수정 대상 파일은 **편집 직전에 반드시 현재 상태를 다시 읽는다** (이전 대화에서 읽은 내용은 stale일 수 있음).
3. **PPTX 주의**: 사용자가 PowerPoint로 직접 고친 pptx는 생성 스크립트(`제안서/제안서_생성.py`)와 불일치할 수 있다. **스크립트 재생성은 pptx의 수작업 편집을 통째로 덮어쓴다** — 재생성 전에 현재 pptx와 스크립트 산출물이 일치하는지 확인하고, 불일치하면 (a) 수작업 변경을 스크립트에 역반영한 뒤 재생성하거나 (b) pptx를 unpack→직접 수정→pack 으로 부분 수정한다. 판단이 어려우면 사용자에게 묻는다.
4. **요청받은 부분만 수정**하고 그 외는 건드리지 않는다. 재생성 방식이라 어쩔 수 없이 다른 부분이 바뀌면 반드시 사전에 알린다.

### 마스터-미러 체제 (제안서 2종)
- **`변동성하베스트펀드_KOSPI200_초안.pptx` = 마스터** — 사용자가 직접 작업.
- **`변동성하베스트펀드_AITop2_초안.pptx` = 미러** — K200의 변경분을 Claude가 **동일 기준으로** 반영.
- 미러 절차: ① git HEAD 대비 K200 현재본 텍스트 diff로 사용자 수정분 파악 → ② AITop2에 python-pptx **부분수정**(unpack/재생성 금지, 디자인·서식 보존, run 단위 치환) → ③ 두 파일 diff로 "펀드 고유 차이(제목·기초자산·σ50/60·지수↔단일종목 위험)"만 남는지 검증.
- 펀드 고유 차이는 미러하지 않고 유지한다. 표지 제목은 서로 달라야 함(예: "변동성 하베스트 펀드 제안서" vs "AI Top2 변동성 하베스트 펀드 제안서").

## 작업 A — 시뮬레이터 실행/검증
대상: `tools/옵션복제_시뮬레이터.html` (자체완결 HTML, 외부 의존성 없음).
1. 정적 서버 기동: 프리뷰 `volfund`(포트 8778) 사용 → `http://localhost:8778/tools/옵션복제_시뮬레이터.html`.
   (설정은 ETF WEB `.claude/launch.json`의 `volfund`. 없으면 `python -m http.server 8778 --directory <변동성펀드>`.)
2. 검증 체크리스트:
   - 콘솔 오류 0.
   - 좌(이론가 BS/Barrier)·우(MC Dynamic Hedging) **병렬 카드** 렌더.
   - **이론가(BS) ≈ MC 복제가치** (정합). 배리어는 MC가 약간 큰 것이 정상(이산 배리어 편의).
   - 트래킹오차 평균 ≈ 0 (거래비용 제외). 손익분포·목표확보확률·VaR 표시.
3. 값 확인은 `preview_eval`로 각 카드 `.stat` 텍스트를 긁어 수치 검증(스크린샷만 믿지 말 것 — 작은 이미지 오독 주의).

## 작업 B — 시뮬레이터 수정 (수식/UI)
`reference.md`의 수식 규약을 준수. 특히:
- 옵션 타입 코드 `'c'`/`'p'` (UI select 값 포함). `'call'`/`'put'` 금지.
- 배리어는 Reiner-Rubinstein "아웃 = 유러피언 − 인". 그릭스는 FD.
- MC: 위험중립 (r−q) 드리프트, 현금 무위험성장+배당(q) 반영, 트래킹오차는 거래비용 제외.
- 부호: 숏풋 보유가치·MC복제가치 음수, "수취 프리미엄"은 + 표기.
수정 후 반드시 작업 A 검증 재수행. 원본은 `tools/옵션복제_시뮬레이터_backup_YYYYMMDD.html`로 백업.

## 작업 C — 백테스트
1. KOSPI200 과거 시계열 확보(지수 값 규칙 준수 — 추정 금지). 데이터 위치·형식은 사용자와 확인.
2. 선정 구조로 회차 롤링 복제 백테스트: 목표달성률(6개월 이내/전체), 회차 손익분포, 대표 사례(달성/미달) 산출.
3. 결과·스크립트는 `백테스트/`에 저장. **제안서 수익률 구조는 이 결과에서만 인용**(창작 금지).

## 작업 D — 제안서 PPTX 작성/갱신
1. **형식 근간**: `변동성매매자문_20260513.pptx` (읽기전용 참조). `pptx` 스킬로 unpack→편집→pack, 또는 python-pptx.
2. 슬라이드 구성(최근 파일 흐름) — `reference.md` 참조.
3. **수익률 구조 = 작업 C 백테스트**, **마케팅 포인트 = 사용자 별도 제공**. 둘 다 임의 창작 금지.
4. 컴플라이언스 문구·"이해를 돕기 위한 예시" 캡션 유지. 지수·수치 출처 표기.
5. 산출물은 `제안서/`에 저장, 원본 편집 시 `*_backup_*`.
6. 검증: `extract-text`로 내용/오타/누락 확인 + soffice로 PDF 변환 후 서브에이전트 시각 QA(overflow/overlap).
